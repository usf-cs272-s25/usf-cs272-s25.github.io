We will use SQLite 1. This deck illustrates two alternative approaches --- ## Install SQLite Widely used, but requires `gcc` ```sh $ go get github.com/mattn/go-sqlite3 ``` ```go import _ "github.com/mattn/go-sqlite3" ``` Pure Go ```sh $ go get modernc.org/sqlite ``` ```go import _ "modernc.org/sqlite" ``` ---

# Using Go [`database/sql`](https://pkg.go.dev/database/sql) --- ## Connect to a Database ```go import "datbase/sql" db, err := sql.Open("sqlite", "example.db") if err != nil // error handling defer db.Close() // operate on the database ``` Under the covers ```sql CREATE DATABASE example.db ``` --- ## Creating Tables ```go result, err := db.Exec(` CREATE TABLE customers( id integer PRIMARY KEY AUTOINCREMENT, name text UNIQUE, street_number integer NOT NULL, street_name text NOT NULL, city text NOT NULL, state varchar(2) NOT NULL, postal_code integer NOT NULL ); `) ``` --- ## type `sql.Result` ```go type Result interface { LastInsertId() (int64, error) RowsAffected() (int64, error) } ``` --- ## Inserting Rows ```go result, err := db.Exec(` INSERT INTO customers(name, street_number, street_name, city, state, postal_code) VALUES("Widgets Inc", 123, "Main St", "San Francisco", "CA", 94117); `) if err != nil // error handling newId, err := result.LastInsertId() if err == nil { fmt.Println("new ID: ", newId) } ``` --- ## Querying ```go zip := 94117 rows, err := db.Query(` SELECT (name, street_number, street_name) FROM customers WHERE postal_code = ? `, zip) if err != nil // error handling for rows.Next() { var custName, streetName string var streetNumber int err = rows.Scan(&custName, &streetNumber, &streetName) if err == nil fmt.Println(custName, streetNumber, streetName) } ``` ---

## Can we unify the SQL world with Go structs? ---

## Object Relational Mapping (ORM) 1. Translate between structs/classes and the comma-separated tuples of SQL 1. Available in many languages, including Java and Go 1. What Go language feature could help us? 1. gorm is a ([fantastic](https://gorm.io/)) ORM for Go --- ## Installing gorm ```sh go get -u gorm.io/gorm go get -u gorm.io/driver/sqlite ``` ```go import ( "gorm.io/gorm" "gorm.io/driver/sqlite" ) ``` --- ## Connect to a Database ```go type Customer struct { gorm.Model // <- what's happening here? Name string StreetNumber int StreetName string //... } db, err := gorm.Open(sqlite.Open("test.db"), &gorm.Config{}) if err != nil { panic("failed to connect database") } // Migrate the schema db.AutoMigrate(&Customer{}) ``` Under the covers ```sql CREATE TABLE customers(...); ``` --- ## Inserting Rows ```go // Remember to pass the address of the struct result := db.Create(&Customer{ Name: "Widgets Inc", StreetNumber: 123, StreetName: "Main St", ... }) if result.Error == nil { fmt.Println(result.RowsAffected) } ``` --- ## Querying ```go var customer Customer result := db.First(&customer) ``` Under the covers ```sql SELECT * FROM customers LIMIT 1; ``` --- ## More Querying ```go cust := Customer{ID: 1} result := db.First(&cust) ``` Under the covers ```sql SELECT * FROM customers WHERE id = 1; ``` --- ## Updating Rows ```go cust := Customer{ID: 1} result := db.Model(&cust).Update("postal_code", 95070) ``` Under the covers ```sql UPDATE customers SET postal_code=95070 WHERE id=1; ``` --- ## Struct tags 1. In Go, a struct member can be followed by a "tag", expressed in raw bytes ```go type Customer struct { ID int `json:"id"` Name string `json:"name"` } ``` --- ## Tags in gorm 1. gorm translates Go's CamelCase names to SQL snake_case names automatically 1. However, `gorm.Model` shows other ways to use Go tags ```go // gorm.Model definition type Model struct { ID uint `gorm:"primaryKey"` CreatedAt time.Time UpdatedAt time.Time DeletedAt gorm.DeletedAt `gorm:"index"` } ``` --- ## Constraints in gorm 1. Primary key and foreign key are created by `db.Migrate()` 1. Other constraints can be added via tags ```go type Customer struct { gorm.Model Name string `gorm:"index;unique;not null"` } ``` --- ## Documentation 1. gorm is extensively documented: [https://gorm.io/docs/index.html](https://gorm.io/docs/index.html) --- ## ORM Full Disclosure 1. ORMs can be convenient, but there are tradeoffs 1. What tradeoffs do you see with gorm-like tools? 1.
